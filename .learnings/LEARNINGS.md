# Learnings Log

This file captures corrections, knowledge gaps, and best practices discovered during sessions. Broadly applicable learnings get promoted to project documentation.

## Format

Each entry:
- **ID:** LRN-YYYYMMDD-NNN
- **Title:** Brief description
- **Category:** `correction` | `knowledge_gap` | `best_practice` | `tool_gotcha` | `workflow_improvement`
- **Priority:** `critical` | `high` | `medium` | `low`
- **Status:** `pending` | `resolved` | `promoted`
- **Area:** (e.g., `frontend`, `backend`, `infra`, `docs`, `config`)
- **Pattern-Key:** (for recurring issues, a stable dedupe key)
- **Recurrence-Count:** (how many times this pattern appeared)
- **First-Seen:** date
- **Last-Seen:** date
- **See Also:** links to related entries
- **Promoted:** (if promoted to CLAUDE.md, AGENTS.md, TOOLS.md, SOUL.md)
- **Description:** Detailed explanation, reproduction steps, concrete fix
